Rest Params
-----------
- The rest parameter syntax (...)
    - Looks like the spread syntax (...) but it is NOT
        - Collects the "rest" of a function's passed argument value(s) into an actual array

    - Unlike the spread syntax that "spreads" data into individual components
        - The rest syntax collects the "rest" of the data into a single parameter variable

    - Every function has an "arguments" object
        - It is an array-like object that HOLDS ALL argument value(s) passed to a function

        - It has a property ".length"
            - But NO support for array methods (e.g. reduce, push, pop, etc.)
            
        - IMPORTANT: The "arguments" object is NOT available on an Arrow function
            - Another reason to use rest parameter 

    - It is placed at the end of the a function's parameter list
        - If a function has multiple named parameter variables 

    - The following are some syntax restrictions
        - A function can ONLY HAVE ONE (1) rest parameter variable
        - Trailing commas are NOT ALLOWED AFTER the rest parameter variable
        - The rest parameter CAN NOT HAVE a default value

    - Reference (https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Functions/rest_parameters)


Syntax:
function f(<params_1>, <params_n>, <...theArgs>) {
  // ...
}
